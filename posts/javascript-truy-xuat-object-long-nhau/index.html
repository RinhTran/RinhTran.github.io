🧩 [JavaScript] Kinh nghiệm truy xuất giá trị trong object lồng nhau

🗓️ October 24, 2025 #JavaScript #Lodash #Thư viện

🔎 Giới thiệu

Chắc hẳn bạn đã nhiều lần phải truy xuất thuộc tính của Object để lấy dữ liệu (API, localStorage, JSON…).
Khi cấu trúc dữ liệu lồng nhau (nested object), việc truy cập trực tiếp rất dễ crash nếu một tầng nào đó bị undefined / null.

Bài viết này tổng hợp các cách an toàn, hiện đại để lấy giá trị từ object lồng nhau, kèm ví dụ cụ thể.

  3. Cách 1: Dùng điều kiện “bảo hộ” (guard checks)
  4. Cách 2: Optional Chaining ?. + Nullish Coalescing ??

1) Ví dụ dữ liệu & truy xuất trực tiếp

Giả sử server trả JSON như sau (được parse thành object):

let response = {
  data: {
    options: {
      name: "Dương Anh Sơn",
      website: "https://vntalking.com"
    }
  }
};

Truy xuất trực tiếp:

const website = response.data.options.website;
console.log(website);
// → https://vntalking.com

✅ Đúng khi mọi tầng đều tồn tại.
❌ Dễ crash nếu response.data hoặc options không có.

2) Vấn đề: thuộc tính bị undefined

Nếu response thay đổi (ví dụ thiếu options):

let response = { data: {} };
// response.data.options === undefined

Truy cập response.data.options.website sẽ throw lỗi vì đang đọc .website của undefined.

3) Cách 1: Dùng điều kiện “bảo hộ” (guard checks)

Cách truyền thống: kiểm tra lần lượt từng tầng trước khi chạm tới giá trị cuối.

let website = "Không biết";
if (response && response.data && response.data.options && response.data.options.website) {
  website = response.data.options.website;
}
console.log(website);
  • Ưu: rõ ràng, không cần thư viện.
  • Nhược: dài dòng, dễ sai sót khi sâu nhiều tầng.

4) Cách 2: Optional Chaining ?. + Nullish Coalescing ??

Từ ES2020, JavaScript hỗ trợ optional chaining và nullish coalescing:

const website = response?.data?.options?.website ?? "Không biết";
console.log(website);
  • ?. dừng truy cập nếu gặp null/undefined, trả về undefined thay vì crash.
  • ?? chỉ dùng default khi vế trái là null hoặc undefined (khác với ||).

So sánh nhanh ?? vs || 

0 ?? 42        // 0        (không thay thế)
0 || 42        // 42       (0 là falsy → bị thay thế)

"" ?? "N/A"    // ""       (giữ nguyên chuỗi rỗng)
"" || "N/A"    // "N/A"    ("" là falsy → bị thay thế)

💡 Dùng ?? khi bạn chỉ muốn thay default cho null/undefined.

5) Cách 3: Dùng Lodash _.get

Nếu dùng Lodash, _.get là “dao đa năng” để đọc sâu:

// npm i lodash
import _ from "lodash";

const website = _.get(response, "data.options.website", "Không biết");
console.log(website);
  • Ưu: ngắn gọn, có default tích hợp.
  • Nhược: thêm phụ thuộc (dependency).

Mẹo: Dùng _.set, _.has, _.unset để ghi/xóa/kiểm tra an toàn theo path.

6) Cách 4: Tự viết helper getIn (thuần JS)

Không muốn cài thư viện? Bạn có thể tự viết hàm nhỏ gọn:

function getIn(obj, path, defaultValue) {
  // path: chuỗi "a.b.c" hoặc mảng ["a","b","c"]
  const keys = Array.isArray(path) ? path : String(path).split(".").filter(Boolean);
  let cur = obj;
  for (const k of keys) {
    if (cur == null) return defaultValue; // null hoặc undefined
    cur = cur[k];
  }
  return cur === undefined ? defaultValue : cur;
}

// Ví dụ
const website = getIn(response, "data.options.website", "Không biết");
console.log(website);
  • Ưu: nhẹ, dễ tùy biến (hỗ trợ cả mảng path).
  • Nhược: viết thêm một chút mã nguồn.

7) Bonus: Xử lý mảng, dynamic path, và default “đẹp”

7.1. Path có mảng (array) 

const res = { data: { users: [{ profile: { name: "Alice" } }] } };
const firstName = res?.data?.users?.[0]?.profile?.name ?? "Ẩn danh";
console.log(firstName);

7.2. Path động (runtime) 

const lang = "vi";
const title = getIn(response, ["data", "options", "i18n", lang, "title"], "Không tiêu đề");

7.3. Default “đẹp” với nullish coalescing 

const count = response?.meta?.count ?? 0;       // số
const isOn  = response?.flags?.enabled ?? false; // boolean
const list  = response?.items ?? [];             // mảng rỗng
const obj   = response?.config ?? {};            // object rỗng

8) TypeScript: An toàn kiểu dữ liệu

Với TypeScript, bạn sẽ được gợi ý kiểu và cảnh báo sớm khi truy cập sai path.

type ApiResponse = {
  data?: {
    options?: {
      name?: string;
      website?: string;
    };
  };
};

declare const response: ApiResponse;

const website: string = response?.data?.options?.website ?? "Không biết";

Helper generic (TS) cho getIn 

// Đơn giản, không infer sâu path -> Tự trả về unknown
function getIn<T>(obj: unknown, path: string | (string | number)[], fallback: T): T {
  const keys = Array.isArray(path) ? path : String(path).split(".").filter(Boolean);
  // @ts-ignore - chấp nhận kiểu linh hoạt
  let cur: any = obj;
  for (const k of keys) {
    if (cur == null) return fallback;
    cur = cur[k];
  }
  return (cur === undefined ? fallback : cur) as T;
}

💡 Kết hợp với schema validation (như Zod, Yup) để kiểm tra dữ liệu từ API trước khi dùng.

9) Kết luận

  • Không truy cập trực tiếp vào path sâu nếu không chắc chắn cấu trúc.
  • Ưu tiên Optional Chaining ?. + Nullish ?? cho code ngắn, an toàn.
  • Cân nhắc Lodash _.get hoặc helper getIn khi cần path động/phức tạp.
  • Với TypeScript, mô tả kiểu rõ ràng để tránh lỗi runtime.
