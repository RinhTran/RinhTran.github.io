1️⃣ Không cho tạo instance với Force Non‑Instantiability

Nếu bạn không muốn class bị khởi tạo bằng từ khóa new, hãy dùng private constructor.
Điều này đặc biệt hữu ích với utility class chỉ chứa các hàm static.

class MovieUtils {
    private MovieUtils() {} // Ngăn không cho tạo instance

    static String titleAndYear(Movie movie) {
        return movie.getTitle() + " (" + movie.getYear() + ")";
    }
}

💡 Mẹo: Các class tiện ích (Utils, Constants, Math,…) nên áp dụng kỹ thuật này để tránh khởi tạo thừa.

2️⃣ Tối ưu với Static Factory Methods

Thay vì sử dụng new trực tiếp, hãy tạo phương thức factory tĩnh (static factory method).
Kỹ thuật này giúp bạn đóng gói logic khởi tạo và có thể trả về subclass hoặc cache instance.

class Movie {
    private String title;

    private Movie(String title) {
        this.title = title;
    }

    public static Movie create(String title) {
        return new Movie(title);
    }
}

Ưu điểm:

  • Không bắt buộc trả về instance mới mỗi lần gọi.
  • Có thể đặt tên dễ hiểu hơn (ví dụ: from(), of(), valueOf(), getInstance()).
  • Hỗ trợ cache và pooling dễ dàng.

⚠️ Lưu ý: Static Factory Method có thể gây khó khăn khi test (mocking).
Nếu gặp vấn đề, cân nhắc dùng Factory Interface hoặc Dependency Injection.

3️⃣ Builder Pattern – Linh hoạt và dễ đọc

Khi một constructor có nhiều hơn 3 tham số, hãy chuyển sang Builder Pattern.
Cách này giúp code dễ đọc, dễ mở rộng và tránh lỗi khi truyền sai thứ tự tham số.

class Movie {
    private final String title;
    private final String genre;

    private Movie(Builder builder) {
        this.title = builder.title;
        this.genre = builder.genre;
    }

    static class Builder {
        private String title;
        private String genre;

        Builder withTitle(String title) {
            this.title = title;
            return this;
        }

        Builder withGenre(String genre) {
            this.genre = genre;
            return this;
        }

        Movie build() {
            return new Movie(this);
        }
    }
}

// Sử dụng:
Movie matrix = new Movie.Builder()
        .withTitle("The Matrix")
        .withGenre("Sci-fi")
        .build();

🧩 Khi nào dùng: Khi class có nhiều tuỳ chọn hoặc các thuộc tính không bắt buộc.

4️⃣ Static Member Classes

Nếu bạn khai báo Inner Class, hãy xem xét biến nó thành Static Class khi có thể.
Điều này giúp giảm rò rỉ bộ nhớ (Memory Leak) vì static class không giữ tham chiếu đến outer class.

class Movie {
    static class MovieAward {
        String name;
        int year;
    }
}

💡 Nếu Inner Class không cần truy cập thuộc tính outer class → luôn dùng static.

5️⃣ Generics ở khắp mọi nơi

Java hỗ trợ Type‑Safety, cho phép hạn chế lỗi kiểu khi biên dịch.
Luôn sử dụng Generics thay vì kiểu Raw Type để code rõ ràng và an toàn hơn.

// ❌ Không nên
List movies = new ArrayList();
movies.add("Hello");
String movie = (String) movies.get(0);

// ✅ Nên
List<String> movies = new ArrayList<>();
movies.add("Hello");
String movie = movies.get(0);

✨ Hàm tổng quát: 

<T> List<T> sort(List<T> input) {
    [...]
    return input;
}

Bounded Wildcards: 

// Đọc dữ liệu – dùng extends
void readList(List<? extends Movie> movieList) { ... }

// Ghi dữ liệu – dùng super
void writeList(List<? super Movie> movieList) { ... }

6️⃣ Trả về Empty thay vì Null

Trả về null khiến người dùng hàm phải kiểm tra null → dễ gây NullPointerException.
Thay vào đó, hãy trả về Collection rỗng.

List<Movie> latestMovies() {
    if (db.query().isEmpty()) {
        return Collections.emptyList();
    }
    return db.query();
}

🛡️ Trả về Collections.emptyList() vừa an toàn, vừa tiết kiệm bộ nhớ.

7️⃣ Không nên nối String bằng toán tử “+”

Toán tử + khi nối nhiều chuỗi sẽ tạo nhiều đối tượng String tạm thời → tốn bộ nhớ.
Hãy dùng StringBuilder hoặc StringBuffer thay thế.

StringBuilder sb = new StringBuilder();
for (Movie movie : movies) {
    sb.append(movie.getTitle());
}
return sb.toString();

🚀 Nếu đang trong vòng lặp, tuyệt đối tránh dùng “+” để nối chuỗi.

8️⃣ Recoverable Exceptions – Xử lý lỗi có thể phục hồi

Thay vì chỉ throw Exception rồi bỏ mặc, hãy xác định rõ lỗi và cho phép xử lý phục hồi.
Điều này giúp ứng dụng thân thiện hơn với người dùng và dễ bảo trì.

List<Movie> latestMovies() throws MoviesNotFoundException {
    if (db.query().isEmpty()) {
        throw new MoviesNotFoundException("Không tìm thấy phim mới");
    }
    return db.query();
}

💬 Hãy chỉ dùng Exception cho tình huống ngoại lệ thực sự, không dùng để điều khiển logic thông thường.

🧩 9️⃣ Tạm kết

Tối ưu mã nguồn Java không chỉ giúp app Android chạy nhanh hơn, mà còn giúp:

  • Mã sạch, dễ đọc, dễ bảo trì.
  • Giảm memory leak, bug runtime.
  • Tăng khả năng mở rộng và test.
Kỹ thuật Lợi ích chính
Force Non‑Instantiability Ngăn khởi tạo thừa
Static Factory Đóng gói logic khởi tạo
Builder Pattern Dễ đọc, dễ mở rộng
Static Inner Class Giảm memory leak
Generics An toàn kiểu dữ liệu
Return Empty Tránh NullPointerException
StringBuilder Cải thiện hiệu năng
Recoverable Exception Xử lý lỗi thân thiện
