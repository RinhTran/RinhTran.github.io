1) Java Interface là gì?

“In its most common form, an interface is a group of related methods with empty bodies.” — Oracle

Diễn nôm: Interface định nghĩa một tập hành vi mà các lớp triển khai (implements) phải cung cấp. Nó không mô tả trạng thái, chỉ mô tả “có thể làm gì”.

Ví dụ “hành vi của mèo”:

interface CatBehaviors {
    void run(int speed);
    void catchMouse(int count);
    void sleep();
}

Một lớp “mèo” sẽ implements các hành vi trên:

class Cat implements CatBehaviors {
    private String furColor;
    @Override public void run(int speed) { /* ... */ }
    @Override public void catchMouse(int count) { /* ... */ }
    @Override public void sleep() { /* ... */ }
}

2) Giống & khác Interface vs Abstract class

Giống

  • Đều mô tả hợp đồng hành vi (contract).
  • Class con phải override method (abstract).

Khác (trọng tâm)

Tiêu chí Interface Abstract class
Mục đích Định nghĩa hành vi “có thể làm” Khai báo bản chất của họ hàng lớp
Trạng thái (fields) Không có state (chỉ public static final constants) Có state (fields non-final), logic chung
Kế thừa Class có thể implements nhiều interface Class chỉ extends 1 abstract class
Method triển khai sẵn Từ Java 8 có default/static methods Có thể có method đã triển khai
Thay đổi hợp đồng Thận trọng: thêm method phá vỡ backward-compat (trừ default) Dễ mở rộng hơn với các method có body

🎯 Quy tắc nhanh:

  • Cần mô tả “khả năng”/hành vi → Interface.
  • Cần chia sẻ trạng thái/logic chung giữa lớp con → Abstract class.

3) Tại sao cần Interface (đa kế thừa kiểu)

Java không hỗ trợ đa kế thừa class, nhưng hỗ trợ implements nhiều interface.
Ví dụ: bạn có CanFly, CanSwim. Một số loài vừa bay vừa bơi → class có thể implements cả hai.

interface CanFly { void fly(); }
interface CanSwim { void swim(); }

class Duck implements CanFly, CanSwim {
    @Override public void fly()  { /* ... */ }
    @Override public void swim() { /* ... */ }
}

4) Đặc điểm & tính năng quan trọng của Interface

  • Không thể khởi tạo: new Interface() ❌
  • Khai báo constants: public static final ngầm định.
  • Từ Java 8: default methods (có body), static methods.
  • Từ Java 9: private methods trong interface (tái sử dụng trong default/static).
  • Nested interfaces: Interface có thể lồng bên trong interface khác/class khác.

5) Ví dụ thực tế: behaviors của động vật

Thiết kế Sở thú: Một số loài bay, một số bơi, một số có tiếng kêu.

abstract class Animal {
    protected String name;
    protected Animal(String name) { this.name = name; }
    abstract String sound();
}

interface CanFly { void fly(); }
interface CanSwim { void swim(); }
interface HasLegs { int legs(); }

class Eagle extends Animal implements CanFly {
    Eagle(String name) { super(name); }
    @Override public String sound() { return "screech"; }
    @Override public void fly() { /* logic bay */ }
}

class Dolphin extends Animal implements CanSwim {
    Dolphin(String name) { super(name); }
    @Override public String sound() { return "click"; }
    @Override public void swim() { /* logic bơi */ }
}

class Duck extends Animal implements CanFly, CanSwim, HasLegs {
    Duck(String name) { super(name); }
    @Override public String sound() { return "quack"; }
    @Override public void fly() { /* ... */ }
    @Override public void swim() { /* ... */ }
    @Override public int legs() { return 2; }
}

💡 Kết hợp abstract class (bản chất + state) với interfaces (khả năng) → thiết kế linh hoạt & rõ ràng.

6) Default method, static method & nested interface

Default method: thêm hành vi mặc định mà không phá vỡ code cũ 

interface CanSleep {
    default void sleep() { System.out.println("Zzz..."); }
}

Static method: tiện ích liên quan đến hợp đồng 

interface MathOp {
    static int clamp(int value, int min, int max) {
        return Math.max(min, Math.min(max, value));
    }
}

Nested interface: gom nhóm các hợp đồng liên quan 

interface Network {
    interface Callback {
        void onSuccess(String body);
        void onError(Throwable t);
    }
    void fetch(String url, Callback cb);
}

⚠️ Tranh chấp default method (diamond): nếu 2 interface có cùng default method, class triển khai phải override để chọn hành vi.

7) Functional Interface & lambda (Java 8+)

Functional Interface: interface có duy nhất 1 abstract method (SAM).
Dùng rất nhiều với lambda / method reference.

@FunctionalInterface
interface Transformer<T, R> {
    R apply(T value);
}

class Demo {
    static String shout(String s) { return s.toUpperCase() + "!"; }

    public static void main(String[] args) {
        Transformer<String, String> t1 = s -> s.trim();
        Transformer<String, String> t2 = Demo::shout;

        System.out.println(t1.apply("  hello ")); // "hello"
        System.out.println(t2.apply("java"));     // "JAVA!"
    }
}

🧠 Các functional interface sẵn có: Runnable, Callable, Supplier, Consumer, Function, Predicate, BiFunction, …

8) Best practices & Pitfalls

✅ Nên

  • Dùng interface để mô tả khả năng/hợp đồng, không nhồi nhét state.
  • Tách nhỏ interface theo SRP (Interface Segregation Principle).
  • Cân nhắc default method khi mở rộng API mà vẫn back‑compat.
  • Đặt tên interface theo hành vi: Readable, Closable, Cacheable, Serializable…
  • Trong Android, interface nhỏ gọn giúp mock/test dễ hơn (DI, unit test).

❌ Tránh

  • Thêm method vào interface public đang dùng rộng rãi (đập vỡ backward‑compat).
  • Lạm dụng interface cho mọi thứ (anti‑pattern anemic interface).
  • Trùng tên default methods từ nhiều interface mà không override để phân giải.
  • Dùng interface lớn kiểu “God Interface” (quá nhiều method).

9) Tổng kết

  • Interface: mô tả hành vi/capabilities, hỗ trợ đa kế thừa kiểu, rất phù hợp để thiết kế API linh hoạt.
  • Abstract class: chia sẻ trạng thái + logic chung giữa các lớp cùng bản chất.
  • Kết hợp đúng cả hai giúp hệ thống dễ mở rộng, dễ test, dễ bảo trì.
  • Tận dụng các tính năng Java 8+: default, static, private method trong interface; functional interface & lambda để viết code gọn & rõ.

✍️ “Interface chỉ là công cụ để thực hiện đa hình — hiểu bản chất để dùng đúng chỗ, đừng thần thánh hoá.”

🧪 Demo nhỏ (chạy nhanh trên console) 

public interface ExtInterface {
    void method1();
    void method2();
}

import java.util.Scanner;
class Edureka implements ExtInterface {
    @Override public void method1() {
        System.out.println("implementation of method1");
        Scanner scanner = new Scanner(System.in);
        System.out.print("Enter number to find square root in Java: ");
        double square = scanner.nextDouble();
        double squareRoot = Math.sqrt(square);
        System.out.printf("Square root of number: %f is : %f%n", square, squareRoot);
    }
    @Override public void method2() {
        System.out.println("implementation of method2");
    }

    public static void main(String[] args) {
        ExtInterface obj = new Edureka();
        obj.method1();
    }
}
