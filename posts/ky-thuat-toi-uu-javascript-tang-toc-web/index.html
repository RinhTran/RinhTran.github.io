⚡️ Kỹ thuật tối ưu **JavaScript** nhằm tăng tốc độ web gấp 2 lần

🗓️ October 24, 2025 #JavaScript #Lodash #Thư viện

🚀 Giới thiệu

Khi ứng dụng của bạn ngày càng phát triển với vô số tính năng được thêm vào, hiệu năng (performance) luôn là vấn đề đau đầu.
Bạn có thể gặp phải:

  • Animation không mượt.
  • Scroll bị giật.
  • UI treo tạm thời.

Tất cả khiến trải nghiệm người dùng tệ đi rõ rệt.

Để giải quyết vấn đề này, bài viết sẽ hướng dẫn bạn cách tối ưu JavaScript để tăng tốc độ web — có thể gấp 2 lần nếu làm đúng!

1️⃣ FPS là gì?

FPS (Frames Per Second) — số khung hình hiển thị mỗi giây.
Trong lập trình web, con số 60 FPS là tiêu chuẩn vàng để app được coi là mượt mà.

  • 1 giây = 1000 ms
  • 1000 / 60 = ~16.6 ms → bạn chỉ có 16.6 mili giây để render 1 khung hình.

Nếu code xử lý lâu hơn 16ms, khung hình bị chậm → người dùng cảm thấy giật lag.
Khi FPS xuống dưới 30 FPS, UI bắt đầu “đơ” rõ rệt.

🎯 Mục tiêu: Giữ cho toàn bộ tác vụ xử lý mỗi frame dưới 16ms (hoặc ít nhất dưới 30ms).

2️⃣ Cách tính Performance để tối ưu JavaScript

🧮 Công thức cơ bản: 

1000ms / 60 FPS = ~16.6ms / frame
1000ms / 30 FPS = ~33.3ms / frame

Nếu thời gian xử lý 1 khung hình vượt quá 33ms, trang web của bạn sẽ rớt FPS đáng kể.

⚙️ Cấu trúc của JavaScript Runtime

JavaScript runtime bao gồm 3 thành phần chính:

  1. Call Stack – nơi chứa các lời gọi hàm đang chạy.
  2. Render Queue – hàng đợi để vẽ UI.
  3. Callback Queue – chứa các hàm chờ được gọi lại (setTimeout, event…).

Trình tự ưu tiên:
Call Stack > Render Queue > Callback Queue

💡 Nếu Call Stack bị “nghẽn” (do code nặng, vòng lặp lớn, xử lý sync…), trình duyệt không thể render đúng lúc ⇒ UI giật.

3️⃣ Một số gợi ý tối ưu JavaScript

⚙️ Gợi ý 1: Ưu tiên sử dụng API bất đồng bộ (Async)

Các tác vụ nặng nên xử lý bất đồng bộ để không chặn Call Stack.

❌ Ví dụ tệ: 

// API đồng bộ (Sync) – chặn luồng chính
fs.writeFileSync('message.txt', 'Hello Node.js');
console.log('The file has been saved!');

Hàm writeFileSync() sẽ chặn luồng chính cho tới khi ghi file xong → làm đứng UI.

✅ Cách đúng: 

// API bất đồng bộ (Async) – không chặn luồng chính
fs.writeFile('message.txt', 'Hello Node.js', (err) => {
  if (err) throw err;
  console.log('The file has been saved!');
});

⚡ Khi sử dụng async API, Render Queue được ưu tiên vẽ lại giao diện trước khi callback được xử lý, giúp UI mượt mà hơn.

🎞️ Gợi ý 2: Chọn API phù hợp cho Animation

Đừng dùng setTimeout hay setInterval cho animation — chúng không đồng bộ với tốc độ refresh màn hình.

❌ Không nên: 

setTimeout(function() {
  console.log('running async callback');
}, 0);

✅ Nên dùng requestAnimationFrame: 

requestAnimationFrame(function() {
  console.log('running async callback');
});

💡 requestAnimationFrame() đồng bộ với refresh rate của màn hình (thường 60Hz) ⇒ animation trơn tru, tiết kiệm CPU, giảm lag.

4️⃣ Tổng kết

Kỹ thuật Mục tiêu Hiệu quả
✅ Dùng API Async Giảm nghẽn Call Stack Tăng FPS, mượt UI
✅ Dùng requestAnimationFrame Tối ưu animation Giảm giật, tiết kiệm CPU
✅ Giữ mỗi frame < 16ms Duy trì 60 FPS Trải nghiệm tốt nhất

🎯 Tóm gọn:

  • Tách các tác vụ nặng ra khỏi Call Stack.
  • Ưu tiên async, tránh blocking code.
  • Đồng bộ animation với refresh rate.
  • Giữ FPS ≥ 60 để trải nghiệm mượt mà như app native.
