    ⚛️ RxJava – Thời đại *Reactive Programming* trong ứng dụng Android

    ⚛️ RxJava – Thời đại Reactive Programming trong ứng dụng Android

    Tác giả: Sơn Dương
    Cập nhật: 2024

    🚀 Mở đầu

    Tư duy hướng sự kiện (event‑driven) vốn quen thuộc ở JavaScript/Node.js nay đã trở thành dòng chảy chủ đạo trên Android nhờ RxJava/RxAndroid.
    Reactive giúp bạn xử lý bất đồng bộ, luồng dữ liệu (streams) và biến đổi dữ liệu (operators) một cách khai báo, súc tích và dễ kiểm soát hơn AsyncTask/Callback thuần.

    Bài viết này là một hướng dẫn từ nền tảng đến thực chiến về RxJava cho Android.

    1) RxJava là gì?

    RxJava là thư viện hiện thực Reactive Extensions (Rx) cho Java, giúp tạo và xử lý streams sự kiện/dữ liệu theo cách bất đồng bộ.
    RxAndroid bổ sung các tiện ích riêng cho Android (như AndroidSchedulers.mainThread()).

    Mô hình cốt lõi tuân theo Observer Pattern: Observable/Flowable phát dữ liệu, Observer đăng ký nhận và xử lý.

    2) Trước RxJava: AsyncTask & giới hạn

    AsyncTask từng là lựa chọn quen thuộc để chạy nền và cập nhật UI sau khi xong việc. Tuy nhiên, nó có nhiều hạn chế:

    • Khó quản lý đa luồng và chuỗi tác vụ.
    • Không có cơ chế biến đổi/lỗi mạch lạc (error handling rời rạc).
    • Dễ memory leak nếu giữ tham chiếu Context sai.
    • Khó test và tái sử dụng.

    Reactive ra đời để giải quyết những vấn đề này bằng cách trừu tượng hóa bất đồng bộ thành chuỗi operators rõ ràng.

    3) Các khái niệm cốt lõi: Observable, Observer, Disposable

    • Observable/Flowable: nguồn phát ra 0..n phần tử (items) và tín hiệu hoàn thành/lỗi.
    • Observer: đăng ký để nhận dữ liệu thông qua onNext, onError, onComplete.
    • Disposable/CompositeDisposable: đại diện cho việc đăng ký (subscription); cần dispose để tránh rò rỉ.

    Ví dụ khởi tạo đơn giản:

    Observable.just("Tiger", "Lion", "Elephant")
    .subscribe(
        item  -> Log.d("RX", "onNext: " + item),
        error -> Log.e("RX", "onError: " + error),
        ()    -> Log.d("RX", "onComplete")
    );

    4) Schedulers & threading trên Android

    • subscribeOn(...): chỉ định luồng thực thi phần tạo dữ liệu (upstream).
    • observeOn(...): chỉ định luồng nhận kết quả (downstream).
    • AndroidSchedulers.mainThread(): nhận kết quả trên UI Thread.
    • Schedulers.io(): I/O (network, file, DB).
    • Schedulers.computation(): CPU‑bound (tính toán).
    Observable.fromCallable(() -> heavyWork())
    .subscribeOn(Schedulers.computation())
    .observeOn(AndroidSchedulers.mainThread())
    .subscribe(result -> render(result));

    🧠 Ghi nhớ: subscribeOn áp dụng một lần (thường đặt sớm), còn observeOn có thể đổi nhiều lần trong chain.

    5) Operators phổ biến (map, filter, flatMap…)

    • map: biến đổi 1‑1.
    • filter: lọc theo điều kiện.
    • flatMap: biến đổi 1‑n, phẳng hóa streams lồng nhau.
    • switchMap: hủy stream cũ khi có stream mới (hữu ích cho search).
    • debounce, throttleFirst: kiểm soát tần suất sự kiện (input, click).
    • zip, merge, concat: phối hợp nhiều nguồn.
    Observable.just(" tiger ", "lion", "ELEPHANT")
    .map(String::trim)
    .filter(s -> s.length() > 3)
    .map(String::toUpperCase)
    .toList()
    .toObservable()
    .subscribe(list -> Log.d("RX", "Animals: " + list));

    6) Thực chiến: cập nhật RecyclerView bằng RxJava 

    CompositeDisposable bag = new CompositeDisposable();

List<String> animals = Arrays.asList("Tiger", "Lion", "Elephant");

bag.add(
    Observable.just(animals)
        .subscribeOn(Schedulers.io())
        .observeOn(AndroidSchedulers.mainThread())
        .subscribe(
            list -> {
                adapter.updateList(list);
                adapter.notifyDataSetChanged();
            },
            throwable -> {
                Toast.makeText(context, "Lỗi: " + throwable.getMessage(), Toast.LENGTH_SHORT).show();
            }
        )
);

    7) Tích hợp Retrofit + RxJava

    Gradle (ví dụ RxJava 2 / RxAndroid 2):

    implementation "io.reactivex.rxjava2:rxjava:2.x.x"
implementation "io.reactivex.rxjava2:rxandroid:2.x.x"
implementation "com.squareup.retrofit2:adapter-rxjava2:2.x.x"

    Khai báo Retrofit với Rx:

    Retrofit retrofit = new Retrofit.Builder()
    .baseUrl("https://api.example.com/")
    .addConverterFactory(GsonConverterFactory.create())
    .addCallAdapterFactory(RxJava2CallAdapterFactory.create())
    .build();

interface Api {
    @GET("users/{id}")
    Observable<User> getUser(@Path("id") String id);
}

Api api = retrofit.create(Api.class);

bag.add(
    api.getUser("42")
        .subscribeOn(Schedulers.io())
        .observeOn(AndroidSchedulers.mainThread())
        .subscribe(
            user -> ui.showUser(user),
            error -> ui.showError(error.getMessage())
        )
);

    🔁 Với RxJava3, thay thế dependencies/adapter tương ứng (rxjava3, adapter-rxjava3).

    8) Quản lý vòng đời & tránh rò rỉ bộ nhớ

    • Dùng CompositeDisposable và dispose/clear ở onDestroy() (Activity) hoặc onCleared() (ViewModel).
    • Tránh giữ Context trong stream. Nếu cần, dùng Application context an toàn hoặc WeakReference.
    • Ưu tiên cold Observables (tạo mới cho mỗi subscriber) trừ khi cố ý dùng hot (Subjects).
    @Override protected void onDestroy() {
    super.onDestroy();
    bag.clear(); // hoặc bag.dispose();
}

    9) So sánh nhanh với AsyncTask/Callbacks

    Tiêu chí AsyncTask/Callback RxJava
    Chuỗi tác vụ Khó, lồng callback Rõ ràng qua operators
    Xử lý lỗi Rời rạc onError() trong cùng chain
    Threading Tự quản lý subscribeOn/observeOn
    Huỷ tác vụ Thủ công, cồng kềnh Disposable.dispose()
    Testability Khó Dễ mock/test từng operator

    10) Best practices & lưu ý

    • Đặt subscribeOn sớm, observeOn ngay trước đoạn cập nhật UI.
    • Bắt lỗi ở cuối chain (onError hoặc onErrorReturn/ResumeNext).
    • Sử dụng CompositeDisposable ở lớp sở hữu vòng đời.
    • Tránh lạm dụng Subjects; ưu tiên streams thuần (Observable/Flowable).
    • Với tác vụ nặng, cân nhắc backpressure (Flowable thay cho Observable).
    • debounce/throttle cho sự kiện UI nhiều: search, scroll, click.
    • Với MVVM, kết hợp LiveData/StateFlow ở biên giới UI nếu cần.

    11) Tổng kết

    RxJava đem lại cách tiếp cận khai báo, mạch lạc cho xử lý bất đồng bộ trên Android:

    • Code ngắn gọn, dễ đọc, chuỗi hóa thao tác dữ liệu rõ ràng.
    • Threading minh bạch qua Schedulers.
    • Error handling & lifecycle quản trị tốt hơn.

    Khi bạn tiến xa với MVP/MVVM, Rx gần như là mảnh ghép mặc định để xây dựng luồng dữ liệu ổn định và dễ test. Hãy bắt đầu từ những bài toán nhỏ (load list, search, debounce click) rồi mở rộng ra network, caching, và phối hợp nhiều nguồn dữ liệu.

    ✍️ Reactive không chỉ là kỹ thuật, mà là cách tư duy về dòng chảy dữ liệu.
